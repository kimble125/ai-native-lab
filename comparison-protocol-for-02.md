# 04. 비교 실행 프로토콜 — 가이드 주입 & 라운드 설계

> 이 문서는 "세 AI에게 **무엇을 / 어떤 순서로** 주입하고, **어떻게 채점**하는가"의 단일 기준입니다.
> 세 모델 모두 **임시/Incognito 세션**(메모리·과거대화 차단)에서 동일하게 실행합니다.

---

## A. 프롬프팅 가이드는 어떻게 주입하나 — "MD 통일" 방식

### 왜 URL이 아니라 MD인가 (공정성 검증 결과, 2026-06)
| 가이드 | URL 본문 읽힘? | 비고 |
|---|---|---|
| Claude | ✅ 읽힘 | 프로젝트에 905줄 MD로 이미 보관 중 |
| GPT (OpenAI) | ❌ 메뉴만 | 본문이 JavaScript 렌더링 → 자동 추출 시 네비게이션만 잡힘 |
| Gemini | ❌ 메뉴만 | 동일하게 JS 렌더링 |

→ **URL로 주입하면 Claude만 전문을 받고 나머지 둘은 깨진 페이지를 받는** 불공정이 생깁니다.
따라서 **세 가이드 모두 "본문을 MD로 고정한 뒤 텍스트로 붙여넣기"** 로 입력 채널을 통일합니다.
(부수 효과: ① 재현성 — 페이지가 바뀌어도 동일 입력 보존 ② 통제 — 불필요한 API/SDK 챕터를 잘라내고 '프롬프트 원칙'만 남길 수 있음.)

### 준비 절차 (각 가이드 공통)
1. 브라우저에서 해당 가이드 페이지를 연다.
2. **프롬프트 작성 원칙에 해당하는 본문만** 복사한다. (API 키·SDK·요금·함수호출 등 무관한 챕터는 제외 → 컨텍스트 절약)
3. 아래 파일에 붙여넣는다.
   - Claude → 프로젝트의 `Claude_Opus_4_8_Prompting_Best_Practices.md`에서 프롬프트 원칙 섹션만 발췌
   - GPT → `guide-gpt.md` (스캐폴드 제공)
   - Gemini → `guide-gemini.md` (스캐폴드 제공)
4. 세 MD의 **분량을 비슷하게** 맞춘다(±20% 내). 한쪽만 길면 그것도 불공정 변수가 됨.

## B. 세션 주입 순서 (각 모델 동일)

```
① [모델 자기 가이드 MD]   "다음은 너(이 모델)를 가장 잘 쓰는 공식 방법이야. 이대로 행동해."
② [raw-brief-input.md]    라운드 A1: 날것 기획을 이해·정리시킴
③ [모델이 만든 정리본 확인] 라운드 A2: "이 브리핑을 '네가 실행하기 가장 좋은 형태'로 다시 다듬어"
④ [system-prompt A/B]      작성 규칙(매일 일지 / 주간 초안)
⑤ [dummy-data 해당분]      입력 데이터
⑥ [환각·검증 문항]         사실/통화 정합성 체크
```

핵심: **가이드(①)를 세션 맨 앞에 둔다.** 그래야 A2(자기 최적화)에서 모델이 *자기 가이드를 실제로 적용*하는지 볼 수 있음.

> 공정성 노트: 가이드를 "각자 자기 것"으로 주는 건 의도된 설계. 실무에서도 모델마다 그 회사 가이드를 적용해 쓰는 게 정상이므로, 이건 **"모델 + 그 모델 공식 가이드 적용" 상태**를 비교하는 더 현실적인 테스트임.

---

## C. 라운드 구조 — A1 / A2 / B (3단)

| 라운드 | 입력 | 산출물 | 측정하는 능력 |
|---|---|---|---|
| **A1 이해·정리** | `raw-brief-input.md`(ⓑ) | 깔끔한 프로젝트 브리핑 | 비정형 정보를 구조화·요약하는 능력 |
| **A2 자기 최적화** | A1 산출물 + 자기 가이드(①) | 모델 자신에게 최적화된 브리핑/프롬프트 | **'AI 자기 이해'** — 자기 강점에 맞게 입력을 재설계하는가 |
| **B 실무** | A2 산출물 + 더미데이터 | 매일 일지 / 주간 초안 | 정해진 포맷으로 정확·일관되게 산출하는 능력 |

- **A와 B는 별개 능력이 아니라 연결된 파이프라인**: A2의 산출물(자기 최적화 브리핑)이 그대로 B의 입력이 됨. (네가 제안한 흐름 그대로)
- 기본 토대(00-brief 골격)는 고정, **스타일만 각 모델이 변형** → A2에서 측정.

## D. 채점 축 (가이드 주입 효과 반영 옵션)

### 옵션 1 — 간단 (권장, 이번 과제용)
- **B(실무)** 만 1~5점 정량 채점: ① 사실 정확성(환각) ② 데이터·통화 정합성 ③ 포맷 준수 ④ 구조화 품질(+SEO) ⑤ 반복 일관성
- **A1·A2** 는 가벼운 루브릭(상/중/하)으로 정성 평가. A2에 "자기 가이드를 실제로 적용했는가"를 한 줄 메모.
- 가이드는 세 모델 모두 **항상 ON** (공정) → '가이드 유무' 비교는 안 함.

### 옵션 2 — 정교 (포트폴리오 심화용, 작업량 ↑)
- 옵션 1의 5축 + **6축: 자기 최적화 품질(AI 자기 이해)** 을 정량화.
- 추가로 각 모델을 **가이드 ON / OFF 두 번** 실행해 "가이드가 결과를 얼마나 바꾸는가"를 7번째 관찰값으로 기록.
- 장점: "프롬프트 가이드 적용이 실제로 효과 있나?"라는 분석가다운 질문에 데이터로 답할 수 있음.

> **추천**: 이번 학원 과제는 옵션 1로 제출하고, 옵션 2는 README에 "향후 확장"으로 명시. (과제가 복잡하지 않다는 네 메모 반영)

## E. 가이드 포함을 평가항목에 넣을지 — 결론
- 넣되 **'유무 비교'(옵션 2)는 선택**. 기본은 "세 모델 모두 자기 가이드 ON"으로 통일하고,
- A2 루브릭에 **"공식 가이드의 어떤 원칙을 실제로 적용했는지 1~2줄"** 만 기록 → 적은 비용으로 '자기 이해' 신호를 남김.
