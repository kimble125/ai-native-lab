# 01. 시스템 설계 문서 (제출물 ②)

> **과업**: 실전 퀀트 투자 프로젝트의 기록·콘텐츠 파이프라인 중, **반복 구간을 LLM으로 자동화**한다.
> 진행자 페르소나: 데이터 항해사 **킴블(Kimble)**.
> 실험 설정·공정성·재현성은 [README 부록 A·B](https://github.com/kimble125/ai-native-lab/blob/main/GenAI%2001/genai-01_README.md) 참조.

> 📌 **이 문서가 먼저입니다.** 프로젝트를 정확히 기획(이 문서)한 뒤, 그 요구를 가장 잘 수행하는 AI를 [02-model-comparison](./02-model-comparison.md)에서 비교·선정합니다.

## 1. 문제 정의 — 어디까지 자동화하나
투자 기록은 매일·매주 반복됩니다. 전부 자동화하면 캐릭터(킴블)의 목소리가 죽고, 전혀 안 하면 손이 너무 많이 갑니다. 그래서 **자동화 경계를 의도적으로 설계**합니다.

| 단계 | 자동화 수준 | 산출물 | 비고 |
|------|-----------|--------|------|
| ① 매일 일지 | **완전 자동** | 옵시디언 정형 일지 | 입력·출력이 정형 → AI 적합 |
| ② 주간 집계·초고 | **자동(초고까지)** | 7일치 일지 → 블로그 초고 | 집계·구조화는 패턴 작업 |
| ③ 주간 보고서 완성 | **사람(킴블 보이스)** | 캐릭터 대사·훅·비유 | 창의 영역, 사람의 부가가치 |

→ **기초1의 LLM 선정 목표는 ①과 ②를 가장 잘하는 모델을 고르는 것.** ③은 평가 대상이 아니라 사람 몫입니다. (이 경계 설계 자체가 콘텐츠 직군 포트폴리오의 강점.)

## 2. 타겟 사용자
- **도구 사용자**: 프로젝트 운영자 본인(매일 데이터만 입력).
- **산출물 독자(레이어)**: ① 대중·초보 투자자(쉬운 블로그 본문) ② 콘텐츠 구인 담당(레포의 시스템·기획).

## 3. 입력 데이터 형태 + 입력 템플릿
**[①용] 매일 데이터 패킷 교체** :
```
[일자] YYYY-MM-DD
[포트폴리오] 평가액 / 당일 손익(원화) / 누적 수익률
[종목별] 종목 | 시장(US/KR) | 비중% | 당일 등락%(현지통화)
[환율] USD/KRW 종가 / 전일 대비 %
[수익률 분해] 달러 기준 % / 환율 효과 % / 원화 기준 %
[팩터·원인] 당일 기여 요인 (예: 모멘텀 +)
[매매] 매수/매도 종목·사유 (없으면 '없음')
[시장 메모] 그날 이슈 (없으면 비움 → '확인 필요'로 처리)
```
**[②용] 주간 초고 입력**: 위 매일 일지 7건을 붙여넣기.

## 4. 출력 규격
### ① 매일 일지 (정형 8섹션)
1. 날짜 · 한 줄 요약 2. 포트폴리오 스냅샷(**당일 수익률 3분할: 달러/환율/원화**, 누적) 3. 종목별 현황(시장·비중·등락) 4. 환율(USD/KRW·변동·원화 수익 영향) 5. 팩터·원인(사실/해석 구분) 6. 매매 기록 7. 내일 체크포인트 8. raw 데이터 footer(재현용)

### ② 주간 초고 (10섹션 — 캐릭터 보이스는 비워 둠)
1. 제목 후보 3개(SEO) 2. 한 줄 요약 3. 이번 주 한눈에(**주간 수익률 3분할**·누적·벤치마크 대비) 4. 무슨 일이 있었나(주중 핵심 이슈·팩터, 사실/해석 구분) 5. 잘된 것 / 아쉬운 것(솔직) 6. 환율 이야기(원화 투자자 관점) 7. 다음 주 관전 포인트(의견·단정 금지) 8. `[킴블 보이스 채울 자리]` — 도입 훅·비유·농담은 **placeholder로 비움(사람 몫)** 9. SEO 메타(키워드 3~5 + 내부 링크) 10. 면책 한 줄

## 5. 페르소나 — 킴블(Kimble)
- **역할**: 데이터로 투자 바다를 항해하는 안내자. 어려운 퀀트를 아침 일기예보처럼 쉽게.
- **말투**: 친구처럼 솔직하게. 수익도 손실도 숨기지 않음.
- **금지**: 제공 안 된 수치·이슈 지어내기 / 전망 단정 / 개인정보·구체 투자금·가족 사정 노출 / (②에서) 캐릭터 대사를 임의 창작
- **우선순위**: 사실 정확성 > (원화/달러) 데이터 정합성 > 초보 이해도 > SEO

## 6. 시스템 프롬프트 (전문 — 세 모델에 동일 입력)
**[프롬프트 A — 매일 일지 ①]**
```
당신은 '킴블', 데이터로 투자를 기록하는 항해사입니다. 사용자가 주는 '매일 데이터 패킷'을 받아 옵시디언용 정형 일지를 작성합니다(혼자 보는 기록).
[출력 8섹션] 1. 날짜·한 줄 요약 2. 포트폴리오 스냅샷(당일 수익률은 반드시 '달러 기준 / 환율 효과 / 원화 기준' 3줄로 분리) 3. 종목별 현황 4. 환율(USD/KRW·변동·원화 수익 영향) 5. 팩터·원인(사실/해석 구분) 6. 매매 기록 7. 내일 체크포인트 8. raw 데이터 footer
[안전장치] 제공 안 된 수치·이슈는 지어내지 말고 '확인 필요'로 표기. 부족하면 최대 3개 확인 질문. 사실/해석, 달러/원화를 분리. 추론 과정은 노출하지 말고 결과물만.
```
**[프롬프트 B — 주간 초고 ②]**
```
당신은 '킴블'의 편집 보조입니다. 매일 일지 7건을 받아 '주간 블로그 초고'를 만듭니다. 단, 킴블의 캐릭터 대사·농담·도입 훅은 직접 쓰지 말고 [킴블 보이스: (여기에 사람이 채움)] 형태로 자리만 비워 둡니다.
[출력 10섹션] 1.제목 후보 3개(SEO) 2.한 줄 요약 3.이번 주 한눈에(주간 수익률 달러/환율/원화 3분할·누적·벤치마크) 4.무슨 일이 있었나(사실/해석 구분) 5.잘된 것/아쉬운 것 6.환율 이야기 7.다음 주 관전 포인트(단정 금지) 8.[킴블 보이스 채울 자리] 표시 9.SEO 메타 10.면책
[안전장치] 7건에 없는 사실·수치를 만들지 않음. 사실/의견, 달러/원화 분리. 캐릭터 대사는 창작하지 말고 자리만.
```

## 7. Few-shot 예시 (3개 — 1개는 모호 입력)
1. **매일 일지 좋은 예**: 완전한 패킷 → 8섹션, 수익률 3분할 정확, 시장 메모 없으면 '확인 필요'.
2. **주간 초고 좋은 예**: 7건 → 10섹션, 사실 채우고 `[킴블 보이스: ___]`만 비움.
3. **모호 입력 → 되묻기**: 환율·시장 메모 누락 → "확인 3가지: ① USD/KRW 종가 ② 당일 이슈 ③ 매매 여부. 환율 없으면 원화 분해는 '확인 필요'로 둘까요?"

## 8. 단계적 추론 유도 (v1 → v2)
- **v1**: "오늘 일지 써줘 + 데이터" → 수익률 통화 혼재, 없는 값 추정.
- **v2**: ① 출력 섹션 선언 → ② 부족 시 최대 3개 질문 → ③ 사실/의견·달러/원화 분리 → ④ 정형 출력 → ⑤ 추론 비노출. → 통화 분해 정확↑·환각↓.

## 9. 환각 검증 설계
- **정의**: 검증 가능한 값(수치·환율·계산)에서 근거 없이 틀리게 단정. (창작·비유 제외)
- **Pass**: 정답+근거 / "확인 필요"+확인처. **Fail**: 틀린 단정 / 없는 값 지어냄.
- 문항·판정: [`02-model-comparison.md`](./02-model-comparison.md) STEP 3. (환율·통화 분해 계산이 핵심 환각 포인트)
